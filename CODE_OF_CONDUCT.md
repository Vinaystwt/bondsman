# Code of conduct

## Our pledge

We want Bondsman to be a constructive and respectful project for builders, reviewers, and security researchers.

## Expected behavior

- Be respectful and direct.
- Assume good intent while still requiring evidence for technical claims.
- Keep reviews focused on the code, contracts, docs, or operational risk.
- Report security issues privately.

## Unacceptable behavior

- Harassment, threats, or personal attacks.
- Publishing private keys, credentials, or exploit details before a fix is available.
- Misrepresenting testnet assets or demo flows as production funds.

## Enforcement

Project maintainers may edit, hide, or remove comments and contributions that violate this policy.
